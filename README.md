מדריך למשתמש: ניהול ועריכת עמוד הנחיתה "סרנדיפיטי" ✨
העמוד שלך בנוי בטכנולוגיית React ומשתמש בספריית העיצוב Tailwind CSS. המדריך הזה יעזור לך להבין "מי נגד מי" בקוד ואיך להפוך אותו לשלך במינימום מאמץ.
1. מבנה הקוד (איפה כל דבר נמצא?)
הקוד מחולק ל"בלוקים" שנקראים section. כל בלוק כזה הוא חלק אחר בעמוד הנחיתה:
חלק 1: Hero - הכותרת הראשית עם אפקט הגרדיאנט והכפתורים.
חלק 2: הצורך (למה הארגון שלכם צריך את זה?) - פסקת הסבר קצרה.
חלק 3: מה קורה במפגש - 3 כרטיסיות (דיוק פנימי, חיבור חיצוני, פתיחת הזדמנויות).
חלק 4: גלריה - תמונות האווירה.
חלק 5: התאמה לקהלים - הכרטיסיות הלבנות עם הקהלים השונים (צעירים, נשים וכו').
חלק 6: מי אנחנו (About) - המקום לתמונות ולטקסט האישי שלך ושל שותפתך.
חלק 7: ציטוט - המשפט הגדול על הרקע הכחול הכהה.
חלק 8: טופס צור קשר (Contact) - המקום שבו המשתמשים משאירים פרטים.
2. איך לשנות טקסטים?
שינוי הטקסט הוא החלק הכי פשוט.
חפשי בקוד את הטקסט שמופיע כרגע (למשל: "דיוק פנימי").
החליפי את המילים שבין התגים (למשל: <h3>דיוק פנימי</h3>) בטקסט החדש שלך.
שימי לב: אל תמחקי את סימני ה- < > או הגרשיים, הם שומרים על המבנה.
3. איך להחליף תמונות?
חפשי בקוד את המילה img. לידה תראי מאפיין שנקרא src.
מה לעשות: החליפי את הקישור שנמצא בתוך הגרשיים (שמתחיל ב-https://) בקישור לתמונה שלך.
איפה להשיג קישור? את יכולה להעלות תמונות ל-Google Drive (ולעשות אותן ציבוריות), ל-Dropbox, או להשתמש באתרים כמו Cloudinary.
4. איך לשנות צבעים? (קסמי ה-Tailwind)
העיצוב מבוסס על שמות של צבעים. אם תראי מילה כמו bg-indigo-600, זה אומר:
bg = רקע (Background).
indigo = הצבע הכחול-סגול.
600 = רמת הכהות (מ-50 הכי בהיר עד 950 הכי כהה).
רוצה לשנות צבע? פשוט חפשי את המילה indigo והחליפי אותה ב:
rose (ורוד-אדמדם)
emerald (ירוק)
amber (צהוב-כתום)
slate (אפור יוקרתי)
5. ניהול הטופס (יצירת קשר)
הטופס שבנינו כולל את השדות: שם, ארגון, תפקיד, טלפון, מייל והודעה חופשית.
כל הכפתורים בעמוד מוגדרים "לקפוץ" לתחתית העמוד ברגע שלוחצים עליהם בזכות הפונקציה scrollToContact.
חיבור למייל: כרגע הטופס הוא עיצובי. כדי לקבל את הפרטים למייל, תצטרכי להשתמש בשירות חיצוני פשוט (כמו Formspree) ולהדביק את הכתובת שלהם בתוך הקוד בתיאום עם איש מקצוע, או להשתמש במערכת ניהול אתרים.
6. טיפים לעבודה נכונה
גיבוי: לפני שאת עושה שינוי גדול, תמיד תשמרי עותק של הקוד שעבד.
תצוגה מקדימה: אחרי כל שינוי קטן, הסתכלי ב"Preview" כדי לראות שהכל נראה טוב.
נייד: העמוד מותאם אוטומטית לנייד. אם משהו נראה מוזר, ודאי שלא מחקת בטעות Classes שמתחילים ב-md: (אלו פקודות למסכים גדולים).
בהצלחה! שהסרנדיפיטי תלווה כל לחיצה על העמוד שלך. ✨
